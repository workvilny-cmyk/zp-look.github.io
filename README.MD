# 🔐 ZP-Lock — Landing Page для службы аварийного открытия замков

Современный, высокопроизводительный Landing Page для локального сервиса по аварийному открытию замков в Запорожье. Сайт оптимизирован для мобильных устройств, так как большинство целевых запросов поступает от пользователей в экстренных ситуациях.

## 🚀 Особенности проекта

* **Dark Mode Design**: Стильная темная тема, снижающая нагрузку на зрение и выглядящая профессионально.
* **Mobile First**: Особое внимание уделено удобству использования на смартфонах — крупные кнопки вызова и плавающая кнопка (FAB) для быстрой связи.
* **Высокая скорость загрузки**: Чистый HTML/CSS без тяжелых фреймворков. Использование современных шрифтов (Inter) и иконок FontAwesome через CDN.
* **Glassmorphism Effects**: Элементы интерфейса (header) используют эффект матового стекла для современного визуального восприятия.
* **SEO-оптимизация**: Настроены мета-теги, семантическая верстка и высокая контрастность текста для поисковых систем.

## 🛠 Технологический стек

Проект реализован на "чистом" стеке для обеспечения максимальной производительности:
* **HTML5**: Семантическая структура.
* **CSS3**: Flexbox, CSS Grid для сетки услуг, кастомные переменные (CSS Variables) и ключевые анимации (keyframes).
* **Google Fonts**: Шрифт Inter для отличной читаемости.
* **FontAwesome**: Векторные иконки.

## 📋 Структура страницы

1.  **Header**: С логотипом, динамическим статусом работы («Зараз працюємо») и кликабельным номером телефона.
2.  **Hero Section**: Главный оффер с четким призывом к действию (CTA) и фоновым изображением.
3.  **Features**: Блок преимуществ (скорость приезда, сохранность имущества, график 24/7).
4.  **Services Grid**: Сетка услуг с карточками, имеющими эффект увеличения при наведении (hover).
5.  **Footer**: Финальный блок с крупным номером телефона и контактной информацией.

## 🔧 Как использовать

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/ваш-аккаунт/zp-lock.git
   ```
2. Откройте файл `index.html` в любом браузере.
3. Для корректного отображения изображений в блоке услуг убедитесь, что файлы `door.jpg`, `avto.jpg`, `self.jpg` и `other.jpg` находятся в корневой папке проекта.
